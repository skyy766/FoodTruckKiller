# 关卡设计文档 — DayLevel_01「第一笔生意」

> 项目：《餐车杀手 Food Truck Killer》  
> 作者：A3 / level-designer  
> 关卡代号：DayLevel_01  
> 状态：MVP 初稿 v1.0

---

## 1. 关卡概述

玩家第一次出摊的白关。地图紧凑（60×40 tile），四类区域围绕一辆固定餐车展开：白天玩家在 **主干道** 招揽顾客、在 **餐车经营区** 烹饪赚钱；当累计赚到 50 元后，第一个暗杀目标出现，玩家需将其用 **诱饵餐** 诱至 **暗巷** 或 **侧巷** 处决、处理尸体并躲避警察与卫生检查员。

单关时长 `480s`（见 `gameconfig.json`）。

---

## 2. 坐标系与尺寸

- **地图尺寸**：60（宽）× 40（高）tile。
- **坐标系**：原点 `(0,0)` 在左上角，`x` 向右递增，`y` 向下递增。下文所有坐标均为 `(x, y)` tile 坐标。
- **世界单位换算**：`1 tile = 1/32 世界单位`（即 32 px，假设 PPU=32）。地图世界尺寸 = `60/32 × 40/32 ≈ 1.875 × 1.25` 世界单位。
- **方向约定**：北 = y 减小（上），南 = y 增大（下），东 = x 增大，西 = x 减小。

## 5. 关键点坐标表

### 5.1 顾客生成点与排队点

| 名称 | 坐标 (x,y) | 说明 |
| --- | --- | --- |
| Spawn_W | (2, 19) | 西端主干道入口，主要顾客源 |
| Spawn_E | (57, 19) | 东端主干道入口 |
| Queue_1 | (28, 22) | 餐车窗口前排第 1 位 |
| Queue_2 | (29, 22) | 排队第 2 位 |
| Queue_3 | (30, 22) | 排队第 3 位（正对窗口） |

- 顾客动线：`Spawn → 沿主干道(y17~22) → Queue_1/2/3 → 窗口(30,24)取餐 → 离场`。
- 窗口（餐车北面服务口）位于 `(30, 24)`，即 `y24` 卡车北墙中央。

### 5.2 警察巡逻路径节点

巡逻采用 **环形往返**：`P1 → P2 → P3 → P4 → P5 → P1`，沿主干道南北两侧人行道绕餐车街区一圈。

| 节点 | 坐标 (x,y) | 位置 |
| --- | --- | --- |
| P1 | (8, 17) | 西北角，暗巷口对面 |
| P2 | (30, 17) | 正对餐车窗口北侧（高压区） |
| P3 | (52, 17) | 东北角，侧巷入口上方 |
| P4 | (52, 23) | 东南角，侧巷入口下方 |
| P5 | (8, 23) | 西南角 |

- 巡逻速度 `2 tile/s`，单圈约 `2×(44+6)/2 ≈ 50s`（一圈总长 100 tile）。
- 响应警报时切换为 `4 tile/s` 直奔最后目击点。
- 视野锥：`range=5, angle=60°`（见 `gameconfig.json`），朝行进方向。
- **设计意图**：P2 经过窗口正北方，玩家在窗口交易时若当场行凶极易被发现；需把握 P2→P3 的窗口期把目标拖入暗巷。

## 6. 区域设计要点

### 6.1 餐车经营区（Truck）
- 卡车占 `x26~35, y24~28`，南墙 `(30,28)` 为门，北墙 `(30,24)` 为服务窗口。
- 内部 `C` 烹饪台、`K` 冷库相邻，玩家转身即可藏尸/取材。
- 卡车四壁遮挡外部视线，是玩家唯一的「安全屋」，但卫生检查员会从南门进入。

### 6.2 主干道（MainRoad）
- `y16~23` 全幅可通行，是顾客与警察的共享通道。
- 顾客从东西两端 `S` 进入，沿 `y19` 行进至 `y22` 排队。
- 警察沿 `y17` 与 `y23` 人行道巡逻，形成十字交叉视野覆盖。

### 6.3 暗巷（DarkAlley）
- `x4~8, y1~15`，北端 `y1` 为死胡同，唯一出口在南端接主干道 `(6,16)`。
- 巷内布置 `@` 诱杀点、`X` 丢弃点、`M` 绞肉机，形成「杀人→处理→销毁」闭环。
- 风险：出口距 P1`(8,17)` 仅 2 格，携尸出巷时若 P1 在岗必被发现。

### 6.4 侧巷（SideAlley）
- `x51~57, y24~38`，北端开口于主干道广场 `y24`，南端 `y39` 封闭。
- `B` 广告牌与 `G` 煤气罐提供两种环境击杀，可伪装成意外。
- 风险：P3`(52,17)`/P4`(52,23)` 在入口上方往返，进巷时机需卡 P3→P4 间隙。

## 7. 卫生检查员动线

- 每 `inspectorIntervalSec=120s` 触发一次（首次在第 120s）。
- 从 `I(30,39)` 进入，沿走廊 `x29~31` 直上，经餐车南门 `(30,28)` 进入餐车内部检查。
- 检查内容：厨房是否有 `isIllegal=true` 食材（人肉/毒药）、是否有血迹证据、冷库是否有尸体。
- 命中违规：`wanted += 40`，并当场没收违禁食材。
- 玩家可在检查员进入前把违禁品转移到暗巷 `X` 或冷库外的临时点。

## 8. 目标引导与布点

- **t_tony**（喜好汉堡）：点汉堡后，玩家端上 `bait_meal` 诱饵餐可使其离开队伍走向暗巷 `@(6,2)`；建议在 P1 远离时动手。
- **t_lena**（喜好热狗）：点热狗后，诱饵引导其前往侧巷 `&(56,28)` 广告牌下；需先确认 P3/P4 不在入口。
- 目标生成条件：累计收入 ≥ 50 元后从 `Spawn_E` 进入（详见 BalanceDesign.md）。

---

## 9. 失败 / 通关条件（本关）

| 类型 | 条件 |
| --- | --- |
| 通关 | 暗杀 ≥1 个目标 **且** 累计金钱 ≥ 150 **且** `wanted < 100`，至 `dayDurationSec` 结束结算 |
| 失败 | `wanted ≥ 100`（通缉度爆表）/ 被警察当场目击击杀 / 时间到未满足通关条件 |
